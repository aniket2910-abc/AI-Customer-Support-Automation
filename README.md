# 🤖 AI Customer Support Automation

An AI-powered Customer Support Ticket Automation System built using **n8n**, **Airtable**, **Gmail**, and **OpenRouter AI**.

This project automates customer support ticket processing by classifying incoming tickets, assigning priorities using AI, updating Airtable records, and sending automated ticket closure emails.

---

# 🚀 Features

- 📩 Automatic customer ticket processing
- 🧠 AI-based priority detection (High / Medium / Low)
- 📊 Automatic Airtable record updates
- 📧 Automatic ticket closure email
- 🔄 End-to-end workflow automation
- ⚡ Reduces manual support effort

---

# 🛠 Tech Stack

- n8n
- Airtable
- Gmail
- OpenRouter AI
- JavaScript

---

# 📂 Workflows

### 🔹 Main Workflow
Receives customer support tickets and stores them in Airtable to start the automation process.

**Example:** A customer submits a "Login Problem" ticket, and it is automatically saved in Airtable.

---

### 🔹 Priority Detection Workflow
Analyzes the ticket content using AI and assigns a priority (High, Medium, or Low).

**Example:** A "Payment Failed" issue is marked as **High Priority**, while a "Login Problem" is marked as **Medium Priority**.

---

### 🔹 Ticket Closing Workflow
Detects when a ticket is marked as **Closed** and automatically sends a confirmation email to the customer.

**Example:** After the support team closes a ticket, the customer receives an email saying, **"Your support ticket has been successfully resolved."**

# ▶️ How It Works

1. Customer submits a support ticket.
2. Ticket data is stored in Airtable.
3. AI analyzes the ticket and assigns a priority.
4. Airtable is automatically updated with the priority.
5. Support team resolves the issue.
6. When the ticket status changes to **Closed**, an automated email is sent to the customer.
